Each resource is stored as a root entity table, a table per component. For resource attributes with compound type we created separate tables (for the sake of storage and consistency separation). Postgresql with default configuration limits length of table names. We didn't want to rebuild postgresql so we shortened table names using the table of abbreviations. Read more about the architecture in README.
